# JavaScript/TypeScript/React QAチェック項目

## エラーハンドリング

- [ ] Promise の未処理拒否（unhandled rejection）が発生していないか
  - [ ] `.then()` チェーンに `.catch()` が繋がれているか
  - [ ] `async/await` の呼び出しを `try-catch` で囲んでいるか
  - [ ] `Promise.all` / `Promise.allSettled` の使い分けが適切か（一部失敗を許容するか否か）
- [ ] エラー境界（Error Boundary）が適切に設置されているか
  - [ ] クラッシュしうるコンポーネントツリーを `ErrorBoundary` で囲んでいるか
  - [ ] フォールバック UI が表示されるか（ユーザーが何も見えない状態にならないか）
  - [ ] `useErrorBoundary` などでイベントハンドラーのエラーも捕捉しているか
- [ ] API エラーのハンドリングが適切か
  - [ ] HTTP エラーステータス（4xx / 5xx）を正しく検出しているか（`fetch` はネットワークエラー以外は reject しない）
  - [ ] エラーをユーザーに通知しているか（トースト・エラーメッセージ等）
  - [ ] ローディング・エラー・成功の各状態を UI に反映しているか

## コード品質

- [ ] コンポーネントの責務が明確か
  - [ ] 1コンポーネントが複数の責務を担っていないか（表示・データ取得・ビジネスロジックが混在していないか）
  - [ ] コンポーネントを分割することで可読性・再利用性を高められるか
- [ ] カスタムフックに切り出せるロジックがないか
  - [ ] コンポーネント内に複数の `useState` + `useEffect` が密集していないか
  - [ ] 同じロジックが複数コンポーネントで重複していないか
- [ ] props drilling が発生していないか
  - [ ] 3階層以上 props を渡し続けていないか
  - [ ] Context API / 状態管理ライブラリ（Redux、Jotai 等）で解決できるか
- [ ] コンポーネントの props インターフェースが適切か
  - [ ] 使われていない props が渡されていないか
  - [ ] 汎用的すぎる `data: any` のような props がないか
  - [ ] children を受け取る場合、`ReactNode` 型が指定されているか

## パフォーマンス

- [ ] 不要な再レンダリングが発生していないか
  - [ ] コンポーネントの props が変わっていないのに毎回レンダリングされていないか
  - [ ] `React.memo` で不要な再レンダリングを防止できるか
  - [ ] 親コンポーネントでインライン定義したオブジェクト/関数を props として渡していないか（毎回新しい参照が生成される）
    - `useMemo` でオブジェクト、`useCallback` で関数の参照を安定させる
- [ ] `useMemo` / `useCallback` の過剰使用をしていないか
  - [ ] 軽量な計算に `useMemo` を使っていないか（オーバーヘッドの方が大きい）
  - [ ] すべての関数に `useCallback` を使っていないか（本当に必要な場所に限定）
  - [ ] メモ化のコストと恩恵を天秤にかけているか
- [ ] 大きなリストのレンダリングを最適化しているか
  - [ ] 数百件以上のリストを全件レンダリングしていないか
  - [ ] ページネーションや無限スクロールで件数を制限できるか

## バグ / ロジックエラー

- [ ] `null` / `undefined` 参照エラーが発生しうる箇所を処理しているか
  - [ ] オプショナルチェーン（`?.`）や Nullish coalescing（`??`）で安全にアクセスしているか
  - [ ] API レスポンスが空の場合の初期値・フォールバックを考慮しているか
- [ ] 非同期処理の競合（Race Condition）が発生しないか
  - [ ] コンポーネントアンマウント後に setState を呼び出していないか
  - [ ] `useEffect` のクリーンアップで進行中のリクエストをキャンセルしているか（`AbortController` の使用）
  - [ ] 連続クリックや高速入力で複数リクエストが並走する場合、最新の結果だけ反映されるか
- [ ] 条件分岐のロジックが正しいか
  - [ ] falsy 値（`0`、`""`、`false`）を意図せず非表示にしていないか（`count && <Component />`）
  - [ ] 配列・オブジェクトの同一性比較を `===` で行っていないか（参照比較になる）
- [ ] イベントハンドラーの登録・解除が適切か
  - [ ] `addEventListener` に対して `removeEventListener` をクリーンアップで呼んでいるか
  - [ ] タイマー（`setTimeout` / `setInterval`）をクリーンアップで `clear` しているか

## セキュリティ

- [ ] XSS（クロスサイトスクリプティング）の対策ができているか
  - [ ] `dangerouslySetInnerHTML` を使っている箇所がないか
    - 使っている場合、値は `DOMPurify` などで適切にサニタイズされているか
  - [ ] `eval()`、`new Function()` を使っていないか
  - [ ] ユーザー入力をそのまま URL に展開していないか（`javascript:` プロトコル）
- [ ] 依存関係のセキュリティを確認しているか
  - [ ] 既知の脆弱性を持つパッケージを使っていないか（`npm audit` で確認）
  - [ ] `package.json` でバージョンをピン留めしているか（`^` や `~` の過剰使用）

## テスト品質

- [ ] コンポーネントの振る舞いをテストしているか（単にレンダリングされることを確認するだけでなく、ユーザーの操作や状態変化に対する反応もテストしているか）
- [ ] API 呼び出しや非同期処理をモックしてテストしているか
- [ ] Presentation コンポーネントでユーザーから見た UI の振る舞いをテストしているか（インテグレーションテストを厚くする）
  - [ ] Hooks のテストは書かれていなくてもよいが、UI の振る舞いを通じて間接的にカバーされているか
- [ ] tsファイルに切り出した関数に十分なユニットテストが書かれているか（ロジックの正確性を担保する）
  - [ ] 正常系、異常系、境界値をカバーしているか

## アクセシビリティ

- [ ] インタラクティブな要素（ボタン、リンク）に意味のある `aria-label` / `aria-labelledby` があるか
- [ ] `onClick` を持つ `div` / `span` がキーボード操作（`onKeyDown`）に対応しているか
  - [ ] 基本は `div` ではなく `button` を使う
- [ ] フォームの `input` に対応する `label` が紐づいているか
- [ ] 画像に `alt` テキストが設定されているか（装飾画像は `alt=""`）
- [ ] フォーカス順序が自然か（`tabIndex` の乱用がないか）

## TypeScript 型安全性

- [ ] `any` の使用を避けているか
  - [ ] `any` を使っている箇所がないか（型安全性を破壊する）
  - [ ] `unknown` を使った上で型ガードで絞り込んでいるか
  - [ ] `as any` でキャストしていないか
- [ ] 型アサーション（`as`）を安易に使っていないか
  - [ ] `as Type` で強制キャストしていないか（実行時エラーの原因になりうる）
  - [ ] `!` の非 null アサーションを安易に使っていないか
  - [ ] 型ガード関数（`isXxx()`）で適切に型を絞り込んでいるか
- [ ] 型定義の品質が適切か
  - [ ] `interface` と `type` の使い分けが一貫しているか
  - [ ] オプショナルプロパティ（`?`）の使いすぎで型が緩くなっていないか
  - [ ] Union 型で `null` / `undefined` を明示的に扱っているか
  - [ ] API レスポンスの型が適切に定義されているか（`unknown` から絞り込む）

## バンドルサイズ

- [ ] 大きなライブラリ全体を import していないか（tree shaking を活用しているか）
  - [ ] 例: `import _ from 'lodash'` ではなく `import debounce from 'lodash/debounce'`
- [ ] コードスプリッティングで初期バンドルサイズを削減できるか
- [ ] `React.lazy` + `Suspense` で画面単位の遅延ロードをしているか
- [ ] 画像・フォントなどのアセットが最適化されているか

## React Hooks のルール

- [ ] Hooks をループ・条件分岐・ネストした関数の中で呼び出していないか
- [ ] `useEffect` の依存配列が正しく設定されているか（過不足がないか）
- [ ] `useEffect` の依存配列を空にして意図的に初回のみ実行しているが、リソースリークがないか（クリーンアップ関数の有無）
- [ ] カスタムフックが単一の責任を持っているか
